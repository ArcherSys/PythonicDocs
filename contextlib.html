contextlib module¶

Utilities for with-statement contexts. See PEP 343.

contextlib.contextmanager(func)[source]¶

@contextmanager decorator.

Typical usage:

@contextmanager def some_generator(<arguments>):

<setup> try:

yield <value>
finally:
<cleanup>

This makes this:

with some_generator(<arguments>) as <variable>:
<body>

equivalent to this:

<setup> try:

<variable> = <value> <body>
finally:
<cleanup>
class contextlib.closing(thing)[source]¶

Bases: object

Context to automatically close something at the end of a block.

Code like this:

with closing(<module>.open(<arguments>)) as f:
<block>

is equivalent to this:

f = <module>.open(<arguments>) try:

<block>
finally:
f.close()

class contextlib.ExitStack[source]¶

Bases: object

Context manager for dynamic management of a stack of exit callbacks

For example:

with ExitStack() as stack:
files = [stack.enter_context(open(fname)) for fname in filenames] # All opened files will automatically be closed at the end of # the with statement, even if attempts to open files later # in the list raise an exception

close()[source]¶

Immediately unwind the context stack

enter_context(cm)[source]¶

Enters the supplied context manager

If successful, also pushes its __exit__ method as a callback and returns the result of the __enter__ method.

pop_all()[source]¶

Preserve the context stack by transferring it to a new instance

push(exit)[source]¶

Registers a callback with the standard __exit__ method signature

Can suppress exceptions the same way __exit__ methods can.

Also accepts any object with an __exit__ method (registering a call to the method instead of the object itself)

class contextlib.suppress(*exceptions)[source]¶

Bases: object

Context manager to suppress specified exceptions

After the exception is suppressed, execution proceeds with the next statement following the with statement.

with suppress(FileNotFoundError):
os.remove(somefile)

# Execution still resumes here if the file was already removed
